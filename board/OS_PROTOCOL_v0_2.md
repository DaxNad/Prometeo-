## Stati di maturità modulo

| Stato | Significato |
|---|---|
| IDEA | concetto definito ma non implementato |
| BUILDING | sviluppo attivo |
| READY LOCAL | stabile in ambiente locale |
| READY REPARTO | utilizzabile in contesto operativo reale |
| READY CAPI | comprensibile e presentabile a responsabili |
| READY CLOUD | funzionante su deploy remoto |
| STABLE | consolidato, non soggetto a modifiche frequenti |

---

## Regole di avanzamento

Un modulo può avanzare solo se:

### READY LOCAL
- funziona senza errori runtime
- API coerenti
- dati leggibili
- nessun blocco tecnico aperto

### READY REPARTO
- output comprensibile a operatore
- terminologia coerente con processo reale
- struttura stabile almeno per 1 ciclo operativo

### READY CAPI
- KPI leggibili
- stato sintetico
- priorità chiare
- output esportabile o consultabile facilmente

### READY CLOUD
- deploy stabile
- endpoint accessibili
- configurazione environment coerente

### STABLE
- nessuna modifica strutturale prevista a breve
- documentazione coerente
- integrazione OS completata
