# PROMETEO — PATTERN LEARNING IMPERATIVE

## STATUS

ACTIVE — PRIMARY PROJECT IMPERATIVE

Questo documento domina roadmap, capability planning, AI integration, planner, runtime, backlog e priorità operative.

## PRIMARY OBJECTIVE

PROMETEO PATTERN LEARNING è l obiettivo centrale, definitivo e prioritario del progetto.

Missione: apprendere dai pattern del Team Leader per ridurre continuamente l arretratezza rispetto alla produzione reale.

## DOMINANCE RULE

Qualunque capability, refactor, planner work, AI integration, architecture work, UI work, dataset work, automation, governance o roadmap deve rispondere prima a questa domanda:

migliora o peggiora il Pattern Learning?

Se NON migliora apprendimento TL, allineamento alla produzione reale, ingest conoscenza reale o qualità runtime decisionale, allora NON è prioritario.

## CANONICAL LEARNING PIPELINE

TL INPUT -> pattern extraction -> candidate pattern -> validation -> pattern registry -> runtime usage -> feedback reale -> reinforcement / correction

## OPERATIONAL TARGETS

PROMETEO deve supportare prioritariamente:

1. fotografia programma produzione giornaliero / settimanale / mensile
2. delega codici a operatori e postazioni
3. supporto target 100 pezzi pro testa
4. apprendimento continuo da nuovi codici progetto, nuove famiglie, nuovi mix turno, nuovi colli di bottiglia e nuove pratiche operative

## PROJECT LAW

Tutto il resto del progetto si deve allineare a PROMETEO PATTERN LEARNING.
