# PROMETEO — TL PATTERN INPUT V1

## STATUS

ACTIVE — INPUT FORMAT FOR PATTERN LEARNING

Formato minimo per trasformare conoscenza TL in pattern PROMETEO.

## PURPOSE

Consentire al Team Leader di inserire pattern produttivi reali in forma semplice, veloce e controllabile.

## CANONICAL INPUT FORMAT

PATTERN TL

Scenario:
Codice/i:
Famiglia:
Postazione/i:
Osservazione reale:
Regola pratica TL:
Quando vale:
Quando NON vale:
Impatto su 100 pz/testa:
Impatto su delega operatori:
Confidenza TL: CERTO / PROBABILE / DA_VERIFICARE
Note:

## EXAMPLE

PATTERN TL

Scenario: codice con due passaggi ZAW
Codice/i: 12511
Famiglia: da verificare
Postazione/i: ZAW1
Osservazione reale: i due passaggi sono entrambi su ZAW1, non ZAW2
Regola pratica TL: non dedurre ZAW2 da doppio passaggio ZAW
Quando vale: codici con doppio innesto rapido semplice
Quando NON vale: giunzioni multiple, manicotti, tre vie o casi confermati ZAW2
Impatto su 100 pz/testa: evita assegnazione errata postazione
Impatto su delega operatori: assegnare a operatore compatibile ZAW1
Confidenza TL: CERTO
Note:

## PROJECT LAW

Ogni pattern TL deve essere acquisito prima come input strutturato, poi trasformato in pattern candidate o regola attiva solo dopo validazione controllata.
